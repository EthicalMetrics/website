Seguridad en EthicalMetrics

La seguridad es un pilar fundamental en EthicalMetrics, protegiendo tanto la integridad de nuestra plataforma como la de los datos anónimos que gestionamos.

Medidas de Seguridad de Datos

Implementamos diversas capas de seguridad para proteger los datos agregados y anónimos que recopilamos, así como para salvaguardar la infraestructura de EthicalMetrics:

  • **Encriptación de Datos:** Los datos se encriptan tanto en tránsito (usando HTTPS/TLS) como en reposo (cuando se almacenan, especialmente con Redis como base de datos).
  • **Anonimización Robusta:** Nuestro diseño inherente asegura que los datos sean anónimos desde el punto de recolección, minimizando el riesgo de exposición de información personal.
  • **Acceso Basado en Token:** Los paneles de control privados por sitio están protegidos mediante `admin_token`, que debe ser manejado con la misma seguridad que una contraseña.

Infraestructura y Auditorías

Nuestra arquitectura de código abierto y nuestro enfoque en la autosuficiencia contribuyen a un entorno más seguro.

  • **Sin Dependencias Externas Innecesarias:** Minimizamos el uso de terceros para reducir la superficie de ataque y el riesgo de vulnerabilidades externas.
  • **Código Abierto para Auditoría:** Al ser un proyecto de código abierto, nuestro código está disponible para que cualquier persona lo audite, lo que fomenta la detección temprana y corrección de posibles vulnerabilidades.
  • **Actualizaciones Constantes:** Mantenemos la plataforma actualizada con las últimas medidas de seguridad y parches.
  • **Reporte de Vulnerabilidades:** Animamos a la comunidad a reportar cualquier vulnerabilidad de seguridad que encuentren. Consulta nuestro repositorio de GitHub para saber cómo.

Para más información sobre cómo manejamos la seguridad de las direcciones IP y el descarte, puedes consultar nuestro documento de Descarte de IP.
